Ritz

A Linux Steam launch wrapper. Set one launch option, and Ritz handles the env vars, gamescope/gamemode wrappers, and launch args for you — from a GUI, per game.

Quickstart Usage guide Write a module

What it solves

Linux gaming means juggling RADV_PERFTEST, gamescope flags, VKD3D_CONFIG, Proton sync backends, frame-gen quirks, and a wall of VAR=value %command% launch options — per game, by hand. Ritz replaces all of that with a single launch option and a settings window. You toggle what you want; Ritz assembles the exact command at launch and stays out of the way.

One launch option

Set ritz %command% once. Everything else is configured in the GUI.

Layered scopes

Settings cascade global → profile → game, each overriding the last.

JSON modules

Every feature is a small JSON file. Edit the bundled ones or drop in your own.

Live preview

The GUI shows the exact assembled launch command as you toggle options.

Quickstart

  1. Build it (Linux only): 
    cargo build --release
  2. Put the binary on your PATH: 
    install -Dm755 target/release/ritz ~/.local/bin/ritz
    (make sure ~/.local/bin is in your $PATH)
  3. Set the Steam launch option — right-click the game → Properties → Launch Options: 
    ritz %command%
  4. Launch. The first time, a wizard asks for a name and an optional profile, then starts the game. After that you get a splash with W launch · E edit · Q cancel.
Settings window: run ritz with no arguments any time to open the full GUI. Dry run: ritz --print %command% prints the assembled command without launching.

Non-Steam games

For non-Steam shortcuts (or anything Ritz can't identify from Steam), give it a stable id:

RITZ_APPID=my-game ritz %command%

Running ritz %command% on an unknown game walks you through setting this up and shows the exact launch option to paste.
